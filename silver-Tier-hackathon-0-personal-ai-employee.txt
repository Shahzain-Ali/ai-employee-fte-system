Line ka matlab:

  Jab aap pehli baar WhatsApp Web use karte ho, toh aapko QR code scan karna padta hai apne phone se. Yeh login process hai.

  Problem: Agar watcher script crash ho jaye ya restart ho, toh normally aapko dubara QR code scan karna padega — jo mushkil hai agar
   aap so rahe ho ya ghar pe nahi ho.

  Solution: Persistent Context

  Playwright mein launch_persistent_context feature hai. Yeh kya karta hai:

  1. Session save karta hai — Jaise browser "Remember me" karta hai
  2. Cookies aur login state ek folder mein store hoti hai

  1. Pehli baar script chalao
  2. WhatsApp Web khulega browser mein
  3. Aap QR code scan karo phone se
  4. Session folder mein login save ho jayegi

  - Script restart → Same session, no QR
  - Computer restart → Same session, no QR
  - Crash ke baad → Same session, no QR

  Bas ek baar scan, phir Playwright khud handle karega. Jab tak session folder delete na ho ya WhatsApp Web logout na kare, QR dubara
   nahi manga jayega.

● Exact Problem — Roman Urdu mein

  Root Cause: googleapis package ka top-level import server startup ko 30+ seconds lag raha tha.

  Timeline:

  1. Python MCP (Failed):
    - FastMCP ka stdio transport properly respond nahi kar raha tha
    - Debug nahi kar paaye kyunki nested Claude session nahi chala sakte
  2. Node.js MCP v1.26.0 (Failed):
    - Server start hota tha but 30s timeout mein initialize nahi hota tha
    - Error: "Connection timed out after 30000ms"
  3. Node.js MCP v0.5.0 (Still Failed):
    - SDK downgrade kiya (Leeza ki version) — same timeout
  4. Root Cause Discovery:
    - Test file banaya: import { google } from 'googleapis';
    - Yeh line 10+ seconds lag rahi thi load hone mein
    - googleapis package BAHUT BHARI hai (171 MB installed)
    - Top-level import = server start hote hi load hota hai
    - Claude Code ka timeout = 30 seconds
    - googleapis load + MCP handshake = 30+ seconds = TIMEOUT
  5. Fix:
  // BEFORE (Top-level — startup pe load)
  import { google } from 'googleapis';

  // AFTER (Lazy — jab tool call ho tab load)
  const { google } = await import('googleapis');

  Kyun kaam kar raha hai ab?

  - Startup: Sirf MCP SDK load hota hai (fast — 1-2 seconds)
  - Handshake: 5 seconds mein complete (well within 30s timeout)
  - Tool call: Tab googleapis load hota hai (user ko wait karna padega first email pe, but server connected hai)

  TL;DR: Heavy package ko startup pe load karna = timeout. Lazy import = startup fast, tool call pe load = success! ✅\
