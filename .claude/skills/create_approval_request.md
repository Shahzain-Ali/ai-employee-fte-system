# Skill: create_approval_request

## Purpose

Create a HITL (Human-in-the-Loop) approval file for any sensitive action.
This skill MUST be called before executing payments, sending emails, or deleting files.
Processing MUST stop until the owner moves the approval file to `Approved/` or `Rejected/`.

## Inputs

Required fields to provide when calling this skill:
- `action_type`: one of `"payment"` | `"email_send"` | `"file_delete"` | `"other_sensitive"`
- `source_action_file`: path to the original action file in `Needs_Action/`
- `reason`: why this action requires human approval
- `details`: object with action-specific fields (see examples below)

## Steps

1. Validate `action_type` is one of the allowed values. If not, reject with error log.
2. Generate filename: `APPROVAL_{action_type}_{YYYYMMDDTHHMMSSZ}.md`
3. Create YAML frontmatter with all required fields
4. Write a human-readable body explaining:
   - **What**: What action is being requested
   - **Why**: Why it requires approval
   - **If Approved**: What happens next
   - **If Rejected**: What happens next
5. Save file to `Pending_Approval/`
6. Call `update_dashboard` skill to show "⚠️ Waiting for Approval"
7. Write log entry with `action_type: "approval_requested"`, `approval_status: "pending"`
8. **STOP processing** — do NOT continue until approval file is moved

## Approval File Format

```markdown
---
id: {UUID}
type: approval_request
action_type: {action_type}
status: pending
source_action_file: {source_action_file}
reason: {reason}
created_at: {ISO timestamp}
---

# Approval Required: {action_type}

**Reason**: {reason}

## Action Details

{formatted details based on action_type}

## Decision Instructions

To **APPROVE**: Move this file to `Approved/`
To **REJECT**: Move this file to `Rejected/`

> ⚠️ Do NOT edit this file. Only move it.
> Processing is paused until you make a decision.
```

## Examples

### Payment Approval

```
action_type: payment
details:
  amount: 150.00
  currency: USD
  recipient: "Vendor Name"
  description: "Invoice #123 for services"
```

### Email Approval

```
action_type: email_send
details:
  to: "recipient@example.com"
  subject: "Meeting request"
  preview: "First 200 characters of email body..."
```

## Output

- Approval file created in `Pending_Approval/`
- Dashboard.md updated to show pending approval
- Processing halted (do not proceed until approved)

## Error Handling

1. **Invalid action_type**: Log error with reason, do NOT create approval file
2. **Pending_Approval/ is full (>10 files)**: Log warning to Dashboard.md
3. **Source file missing**: Log warning, proceed with available information

## Logging

The log entry MUST include:
- `action_type: "approval_requested"`
- `approval_status: "pending"`
- `details.action_type`: the type of sensitive action
- `skill_used: "create_approval_request"`

## Constitution Reference

This skill enforces **Principle II: Human-in-the-Loop (HITL) Safety**.
NEVER bypass this skill for payments, emails, or irreversible actions.
